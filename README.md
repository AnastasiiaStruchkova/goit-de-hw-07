# Домашнє завдання №7: Оркестрація даних за допомогою Apache Airflow

**Опис проєкту**
Цей проєкт реалізує автоматизований **ETL-пайплайн** для обробки олімпійських даних. Основне завдання — вибір випадкового типу медалі, підрахунок кількості відповідних записів у віддаленій базі даних MySQL та збереження результатів у персоналізовану таблицю з перевіркою "свіжості" даних.

---

## Технічний стек
* **Оркестрація:** Apache Airflow 2.9.1
* **Інфраструктура:** Docker, Docker Compose
* **База даних:** MySQL (віддалений сервер GoIT)
* **Мова програмування:** Python 3.9+
* **Бібліотеки Airflow:** `MySqlOperator`, `MySqlHook`, `PythonOperator`, `BranchPythonOperator`

---

## Структура DAG (medal_count_dag)
### Мій DAG реалізує складну логіку розгалуження та моніторингу:

* **create_table:** Автоматичне створення іменної таблиці `anastasiia_struchkova_medals` у схемі `neo_data`.
* **pick_medal:** Випадковий вибір медалі (Gold, Silver, Bronze) та передача значення через **XCom**.
* **branch_task:** Логіка розгалуження для вибору відповідної гілки обробки.
* **count_gold / count_silver / count_bronze:**
    * Підрахунок записів у таблиці `neo_data.athlete_event_results`.
    * Вставка результату в персональну таблицю.
* **sleep_35_sec:** Штучна затримка для тестування сенсора.
* **check_recent_record:** Кастомний `RecentRecordSensor`, який перевіряє, чи було вставлено запис не пізніше ніж **30 секунд** тому.
